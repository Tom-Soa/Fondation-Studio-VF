HTML

La structure — le squelette

Analogie : HTML, c'est les murs, les pièces et les fenêtres d'une maison. Ça dit ce qui existe et où ça se situe — mais pas encore à quoi ça ressemble.

HTML signifie HyperText Markup Language. C'est le langage qui définit le contenu d'une page web : les titres, les paragraphes, les images, les boutons, les liens. Chaque élément est entouré de balises.

Exemple HTML simple
<h1>Fondation Studio</h1> <p>Des sites qui travaillent pour vous.</p> <img src="tom-soa.jpg" alt="Photo de Tom-Soa" /> <a href="contact.html">Nous contacter</a> <button>Demander une maquette</button>

Une balise HTML s'ouvre avec <nom> et se ferme avec </nom>. Tout ce qui est entre les deux est le contenu de cet élément.

BaliseCe que ça créeExemple
<h1> à <h6>Titres (h1 = le plus grand)Titre principal d'une page
<p>Paragraphe de texteDescription d'un service
<a>Lien cliquableLien vers contact.html
<img>ImagePhoto de Tom-Soa
<div>Boîte invisible (conteneur)Section d'une page
<button>Bouton cliquableCTA "Demander une maquette"
<ul> + <li>Liste à pucesListe de fonctionnalités
<input>Champ de formulaireChamp "Votre email"

Dans le site Fondation Studio : chaque fichier index.html, tarifs.html, contact.html est un fichier HTML. Il contient toute la structure de la page — les sections, les textes, les images. C'est lui que le navigateur lit en premier.

Les éléments HTML peuvent avoir des attributs qui donnent des informations supplémentaires :

Attributs — class, id, href, src
<!-- class : groupe des éléments pour le CSS --> <div class="card">Contenu</div> <!-- id : identifie UN seul élément unique --> <section id="hero">...</section> <!-- href : l'adresse d'un lien --> <a href="https://fondationstudio.fr">Site</a> <!-- src : la source d'une image --> <img src="tom-soa.jpg" alt="Tom-Soa" />
CSS

Le style — la peinture et la déco

Analogie : Si HTML est les murs, CSS c'est la peinture, les meubles, les rideaux. C'est ce qui rend tout beau, bien placé, et cohérent.

CSS signifie Cascading Style Sheets. Il dit au navigateur comment afficher chaque élément HTML : couleurs, tailles, espacements, polices, animations.

Une règle CSS se compose de deux parties : un sélecteur (à qui on applique le style) et des propriétés (quoi changer).

Structure d'une règle CSS
/* sélecteur { propriété: valeur; } */ h1 { font-size: 80px; color: #C2410C; /* orange terra */ font-weight: 800; /* très gras */ } .card { /* .card = tous les éléments avec class="card" */ background: #fff; border-radius: 16px; padding: 24px; } #hero { /* #hero = l'élément avec id="hero" */ height: 100vh; /* 100% de la hauteur de l'écran */ }

Les 3 types de sélecteurs les plus courants :

SélecteurCibleExemple HTML
h1Tous les éléments <h1>Chaque titre principal
.cardTous les éléments ayant class="card"Toutes les cartes du site
#heroL'élément unique avec id="hero"La section hero spécifique

Dans le site Fondation Studio, on utilise des variables CSS — des valeurs qu'on définit une seule fois et qu'on réutilise partout :

Variables CSS — utilisées dans tout le site
/* On définit les variables dans :root (la racine du document) */ :root { --midnight: #0F172A; /* bleu nuit */ --terra: #C2410C; /* orange brûlé */ --alabaster:#F9F9F7; /* blanc cassé */ } /* On les utilise avec var(--nom) */ h1 { color: var(--midnight); } .btn { background: var(--terra); } body { background: var(--alabaster); } /* Changer --terra UNE seule fois met tout le site à jour */

Le Box Model : chaque élément HTML est une boîte. De l'intérieur vers l'extérieur : le contenu, puis le padding (espace intérieur), puis la border (bordure), puis le margin (espace extérieur). Comprendre ça, c'est comprendre 80% du positionnement CSS.

JavaScript

Le comportement — ce qui bouge et réagit

Analogie : Si HTML sont les murs et CSS la déco, JavaScript c'est l'électricité, l'ascenseur, les portes automatiques. C'est ce qui rend la page interactive et vivante.

JavaScript (JS) est le seul langage de programmation qui s'exécute directement dans le navigateur. Il permet de réagir aux actions de l'utilisateur : clic, scroll, survol... et de modifier la page en temps réel.

Exemple — réagir à un clic
// On sélectionne un élément HTML const bouton = document.querySelector('.btn'); // On écoute le clic et on exécute du code bouton.addEventListener('click', function() { alert('Merci, on vous recontacte bientôt !'); });

Dans le site Fondation Studio, JavaScript fait :

  • → Le menu burger sur mobile (ouvre/ferme le menu)
  • → Les animations au scroll (les éléments qui apparaissent)
  • → La navigation du diaporama (slide suivant/précédent)
  • → L'effet spotlight sur les cartes (lumière qui suit la souris)
  • → La validation du formulaire de contact
Exemple tiré du diaporama — slide suivant
// On sélectionne tous les slides const slides = document.querySelectorAll('.slide'); let current = 0; // on commence au slide n°0 function slidesSuivant() { slides[current].classList.remove('active'); // cache l'actuel current = current + 1; // passe au suivant slides[current].classList.add('active'); // affiche le suivant } // Quand on appuie sur →, on avance document.addEventListener('keydown', function(e) { if (e.key === 'ArrowRight') { slidesSuivant(); } });

📦

Flexbox

Système CSS pour aligner des éléments horizontalement ou verticalement. Utilisé pour les boutons côte à côte, les cartes en grille, les menus.

📱

Responsive Design

Le site s'adapte à toutes les tailles d'écran. On utilise des @media en CSS pour modifier le style selon la largeur de l'écran.

🔤

clamp()

Une fonction CSS qui crée une taille adaptative : clamp(min, idéal, max). Le texte des titres grossit sur grand écran et rapetisse sur mobile — automatiquement.

🎬

Animations CSS

Les orbes qui bougent en arrière-plan du diapo sont des animations CSS pures. On définit des @keyframes (étapes) et CSS fait la transition tout seul.

🌐

DOM

Document Object Model. C'est la représentation de la page HTML que JavaScript peut lire et modifier en temps réel. Quand JS change un élément, le DOM se met à jour.

⚡

Event Listener

"Écouter un événement". JavaScript attend qu'un truc se passe (clic, touche, scroll) et déclenche une action. C'est la base de toute interactivité.

Comment les 3 langues s'assemblent dans un fichier .html
<!DOCTYPE html> <html> <head> <!-- Le CSS est dans la balise <style> --> <style> .bouton { background: #C2410C; color: #fff; padding: 12px 24px; } </style> </head> <body> <!-- Le HTML est dans <body> --> <button class="bouton" id="mon-btn">Cliquez ici</button> <!-- Le JavaScript est dans <script> --> <script> const btn = document.getElementById('mon-btn'); btn.addEventListener('click', () => { alert('Bonjour !'); }); </script> </body> </html>
